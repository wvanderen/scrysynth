<objective>
Establish the data-driven `NodeCatalogEntry` single source of truth (NODES-01), populate it with the full synthesis chain (NODES-02 oscillator/filter/envelope/LFO/utility, NODES-03 effects), declare per-parameter CV ports (NODES-05), author all v2 SuperCollider SynthDefs with CV-bus args, replace v1's three hardcoded compiler allowlists + two enum-dispatch spots with catalog lookups, allocate control buses for CV wiring, and reject v1 session files with a friendly message (D-09/D-10).

Purpose: This is the single highest-leverage v2 refactor. One Rust-owned catalog table becomes the authority that compiler dispatch, route validation, visual shape, palette, inspector, and ts-rs export all read from — so adding a node becomes a data edit, never a 5-enum + 3-allowlist change. Success criterion #3 (the v1 `unreachable!()` panic becomes a real `Err`) is delivered here.

Output: New `catalog/` module, reshaped `Node` domain model, refactored compiler/synthdefs/visual-compiler, ~13 generated v2 `.scsyndef` binaries, schema bump to v2, two-phase v1 rejection, and existing tests updated to compile/pass against the new model.
</objective>
